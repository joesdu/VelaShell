# VelaShell 交互逻辑与界面规格说明书

> 本文档基于设计稿 `VelaShell-zh.pen` 编写，用于交付给 AI Agent 进行功能实现。
> 目标技术栈：**Avalonia**（UI）+ **自研 VelaDock**（可拖拽/分屏的标签停靠,已替换 Dock.Avalonia;浮动窗口按产品决策禁用）+ 自研 VT 引擎（parser / buffer / emulator / render）。
> 全文以设计稿为唯一视觉基准，凡"应改为…"的条目为对现有设计稿结构的**重构要求**，实现时以本文档为准。

---

## 0. 产品定位与设计基调

VelaShell 是一款面向运维/开发的现代化 SSH/SFTP 终端客户端。核心体验目标：

- **键盘优先**：几乎所有功能都可通过命令面板（Ctrl+P）或快捷键完成。
- **信息密度高但不拥挤**：等宽字体承载数据，UI 字体承载操作，配色克制。
- **浮层化的辅助功能**：文件传输、隧道、资源监控均以可召之即来、用完即走的悬浮面板形式呈现，不占用主终端空间。
- **暗色为主，亮色可切换**：设计稿提供 dark / light 双主题，所有颜色走 token。

## 2. 主界面结构总览

主窗口 1440×900（可自由缩放）。**〔2026-07-12 修订,用户决策〕窗体为无边框自绘样式**：`WindowDecorations="BorderOnly"`，首行为 36px 自绘标题栏（左 = 产品 logo + 名称；右 = 全局功能按钮组 `GQQwj` + 最小化/最大化/关闭），**文字菜单已整体移除**（功能与命令面板重复）。客户区自上而下为：**标题栏 → （侧边栏 ‖ 右区）→ 状态栏**。（下文原生标题栏/菜单栏描述为历史设计,以本段为准：）

```
─────────────── 操作系统原生标题栏（图标 + VelaShell + ─ □ ✕）───────────────
┌─────────────────────────────────────────────────────────────────────────┐
│  菜单栏 Menu Bar（跨整宽 36px）                                             │
│  会话  编辑  操作  搜索  工具  帮助          ……弹性空隙……   [全局功能按钮组 GQQwj] │
├──────────────┬────────────────────────────────────────────────────────────┤
│              │  标签页行：[Tab1][Tab2]…[+]      ……空隙……   [◀][▶][▾]          │
│   侧边栏      ├────────────────────────────────────────────────────────────┤
│  Sidebar     │  终端主区 Terminal（自研 VT 渲染）                            │
│  260px       ├────────────────────────────────────────────────────────────┤
│              │  文件浏览器 / SFTP（可折叠，默认 220px）                       │
├──────────────┴────────────────────────────────────────────────────────────┤
│  状态栏 Status Bar（跨整宽，24px）                                            │
└─────────────────────────────────────────────────────────────────────────┘
```

- **菜单栏为跨整宽独立一行**（`bg-sidebar` 底 + 底部分隔线），位于原生标题栏之下、侧边栏/右区之上。左侧为文字菜单，右侧为全局功能按钮组（原 `GQQwj`，详见 §4A）。
- 菜单栏之下才分「侧边栏 ‖ 右区」两栏；右区顶部为标签页行（详见 §4B）。

- 侧边栏与右区之间、右区上下分栏之间均为可拖拽分隔(自研 VelaDock / GridSplitter)，允许调节宽/高，可整块折叠。
- 终端主区 + 文件区共同构成"一个会话视图"，随激活标签页切换整体内容。

## 3. 侧边栏 Sidebar（宽 260，`bg-sidebar`）

> **窗口标题栏说明**：本应用**沿用操作系统原生标题栏**（由系统绘制应用图标 + "VelaShell" 标题 + 最小化/最大化/关闭按钮），不做自绘窗口。
> 因此**侧边栏顶部不再重复显示应用图标与名称**——原设计稿中的"Logo 栏"（暗色 `tGM2H`、亮色 `4RBrb`）已删除，避免与系统标题栏中的图标+名称重复。侧边栏第一行直接从「工具栏」开始，下方会话资源树相应向上填满释放出的空间。

自上而下：

1. **工具栏（36px，侧边栏首行，`cnUAB`）**：左侧分组标题「资源管理器」（`126Cj`）；右侧两枚按钮——
   - `plus`（`2mkdr`）：新建连接（打开 §13 新建连接弹窗）。
   - `ellipsis`（`oPsSE`）：更多（导入/导出配置、批量操作、折叠全部分组）。
2. **会话资源树（`fill_container`，可滚动）**：分组折叠列表，占据侧边栏中部主要空间。
   - 分组行：可折叠三角 + 分组名（如「生产环境」「测试环境」）+ 计数。
   - 主机行：状态圆点（绿/黄/红）+ 主机名 + 可选标签（如「跳板」`accent-dim` 底）。
   - 当前会话行高亮 `bg-active` 左侧 accent 竖条。
   - 交互：单击选中；**双击=连接并激活标签页**；右键=§12 会话右键菜单；支持拖拽调整分组归属。
3. **会话列表 / 快速连接区（320px，顶部分隔线）**：
   - 头部（36px）：标题「快速连接」+ 折叠按钮。
   - 快速连接输入框（`bg-input`，32px）：可直接输入 `user@host[:port]` 回车连接。
   - 「最近连接」分组标题 + 最近 3 条记录行（`user@host` + 时间），单击回填输入框，双击直接连接。
4. **底部用户栏（40px，顶部分隔线）**：左=当前用户头像 + `root`；右=设置齿轮（打开 §14 设置）+ 主题切换等图标。

## 4A. 菜单栏 Menu Bar（跨整宽 36px，`bg-sidebar` 底 + 底部分隔线）★重构

> 位于原生标题栏之下、侧边栏/右区之上的独立一行。**左侧=文字菜单**，**右侧=全局功能按钮组（原 `GQQwj`，已从终端工具栏迁移到此）**。设计稿节点：菜单栏 `TSiDh`，左侧菜单容器 `DaZfB`，右侧功能容器 `Menu Bar Actions`（内含 `GQQwj`）。

### 4A.1 左侧文字菜单（`DaZfB`）

顺序：**会话 / 编辑 / 操作 / 搜索 / 工具 / 帮助**（Inter 12，`text-secondary`，每项 `padding[0,10]`，hover `bg-hover` + 圆角 3）。单击展开下拉菜单，`Alt` 可键盘访问，`Alt+首字母` 快速打开。各菜单建议内容：

| 菜单 | 典型菜单项 |
|---|---|
| **会话** | 新建 SSH 连接、新建 SFTP、导入/导出配置、最近会话、关闭当前会话、退出 |
| **编辑** | 复制、粘贴、全选、查找、清屏、首选项(设置) |
| **操作** | 连接/断开、重连、分屏、复制会话、组同步开关、广播输入、录制会话 |
| **搜索** | 终端内查找、在文件中查找、跳转到行、命令面板(Ctrl+P) |
| **工具** | 隧道管理、SFTP 文件管理器、运维编排中心、连接诊断、主机信任中心、代码片段、密钥管理 |
| **帮助** | 快捷键一览、文档、检查更新、关于 VelaShell |

> 菜单项与命令面板（§8）、快捷键（§16）三者应共享同一套"命令注册表"，保证入口一致、名称统一。

### 4A.2 右侧全局功能按钮组（原 `GQQwj`）

从终端工具栏迁移到菜单栏最右侧，作为**常用功能快捷入口**（24×22 图标按钮，`gap:4`，hover `bg-hover`，激活 `bg-active`；作用于"当前激活会话/全局"）：

| 图标 | 名称 | 行为 |
|---|---|---|
| `search` | 终端搜索 | 在当前终端缓冲区内查找（打开搜索条，见 §5.3） |
| `copy` | 复制 | 复制当前选区（无选区则灰） |
| `columns-2` | 分屏 | 将当前会话左右/上下分屏（VelaDock 拆分） |
| `route` | **隧道** | 打开 §10 隧道管理面板（★用户指定） |
| `zap` | **快捷命令** | 打开命令面板（§8），或快捷命令菜单 |
| `link-2` | 组同步 | 把当前会话加入/移出输入同步组（激活态高亮） |
| `send` | 广播输入 | 向同步组内所有会话广播本次输入 |

## 4B. 标签页行 Tab Bar（单行 36px，`bg-page` 底）★重构

> 位于右区顶部。**只承载标签页与溢出控制，不再放全局功能按钮**（后者已上移至菜单栏 §4A.2）。设计稿节点：`nunbT`，溢出控制组 `Tab Overflow Controls`（`pZGS4`）。

### 4B.1 布局

```
┌───────────────────────────────────────────────────────────────────────┐
│ [Tab1][Tab2][Tab3 …可横向滚动区…] [+]   ……弹性空隙……          [◀][▶][▾]     │
└───────────────────────────────────────────────────────────────────────┘
     ← 标签滚动容器（clip） →       新建   spacer        左滚动 右滚动 列表下拉
```

- **标签滚动容器**（`clip:true`，`fill_container` 抢占中部）：横向排列全部标签，超出部分裁剪、可横向滚轮滚动；激活标签自动 `ScrollIntoView`。
- **`+` 新建标签**：跟在最后一个标签后。
- **弹性空隙 spacer**：把溢出控制推到最右。
- **溢出控制组 `◀ ▶ ▾`**（右侧，24×24 图标按钮）：
  1. `◀` chevron-left：向左滚动一屏/一个 tab；到最左端禁用置灰。
  2. `▶` chevron-right：向右滚动；到最右端禁用置灰。
  3. `▾` chevron-down：**标签下拉列表**（VS Code 式），列出全部会话名 + 状态点，点击任一项直接激活并滚入可视区；溢出时尤其有用。`◀ ▶` 仅在溢出时可用，`▾` 始终可用。

### 4B.2 单个标签页（Tab）内容

- 结构：状态圆点（`status-connected/connecting/disconnected`，7px）+ 会话名（JetBrains Mono 11）+ 关闭 `x`（12px）。
- 激活态：`bg-terminal` 底 + 顶部 2px accent 竖线 + 文字 `text-primary`。
- 非激活态：`tab-inactive-bg` 底 + 文字 `text-tertiary` + 关闭图标 `text-muted`（hover 才明显）。
- 连接中标签的圆点用黄色；断开用红色。
- 交互：
  - 单击=激活；中键/点 `x`=关闭（有未保存/传输中给二次确认）。
  - 双击标签空白=重命名（inline 编辑）。
  - 拖拽=重排顺序；拖出到边缘=触发 Dock 分屏/浮动窗口。
  - 右键=标签上下文菜单（关闭、关闭其他、关闭右侧、复制会话、重命名、移动到新窗口、加入同步组）。

### 4B.3 标签溢出逻辑（VS Code 式）★用户指定

- 当所有标签总宽 ≤ 容器宽：正常排列，`◀ ▶` 隐藏/禁用，`▾` 仍可用。
- 当溢出：
  1. 容器裁剪，激活标签自动 `ScrollIntoView` 保持可见。
  2. `◀ / ▶` 可点击滚动（到边界时对应按钮禁用置灰）。
  3. 鼠标在容器上横向滚轮可滚动。
  4. `▾` 下拉列表始终列出全部标签，作为溢出时的快速跳转兜底。
- 下拉列表项：状态点 + 会话名 + 右侧当前项勾选标记；支持键盘上下 + 回车选择；可在此直接关闭标签（悬停出现 `x`）。

### 4B.4 悬停唤出资源监控 ★用户指定（详见 §11）

- 鼠标移动到**任意标签页名称**上，**静止悬停 > 400ms** → 在**鼠标当前位置**浮现「系统资源监控面板」。
- 鼠标移出标签（或移入其他标签）→ 面板**自动消失**（建议 150ms 淡出防抖，移入面板本身则保持）。
- 面板展示的是**该标签所对应会话**的资源数据（非当前激活会话），因此可用于快速巡检其他会话。

## 6. 文件浏览器 / SFTP（右区下部，默认 220px，可折叠/拖高）

- **头部（36px）**：左=当前路径面包屑（可点击逐级跳转）+ 刷新；右=视图切换、上传、新建文件夹、隐藏文件开关等图标。
- **列标题（26px，`bg-surface`）**：`文件名(280) | 大小(100) | 权限(120) | 修改时间`。可点击列排序。
- **文件列表（`fill_container`，可滚动）**：每行 28px。图标（文件夹/文件类型）+ 名称 + 大小 + 权限（`drwxr-xr-x`）+ 时间。首行可为 `..` 返回上级。
- 交互：
  - 双击文件夹=进入；双击文件=下载并用默认程序打开（或预览）。
  - 拖拽本地文件到此=**上传**（触发 §9 文件传输组件）；拖拽列表项到本地=**下载**。
  - **同名冲突**：上传/下载遇目标已有同名文件时按设置（§14 文件传输「文件已存在时」）处理——询问（弹确认框：覆盖 or 跳过）/覆盖/跳过/重命名（`file (1).txt`）。
  - 右键=文件上下文菜单（下载、上传、重命名、删除、权限 chmod、复制路径、新建）。
  - 选中态 `bg-hover`；多选（Ctrl/Shift）批量操作。

## 7. 状态栏 Status Bar（跨整宽 24px，`bg-sidebar`）

- **左侧**：`wifi` 图标（连接色）+ `SSH • web-prod-01:22` + `｜` + `Latency: 12ms`（accent）+ `｜` + `↑ 2h 34m`（在线时长）。
- **右侧**：`xterm-256color` ｜ `120×36`（终端尺寸）｜ `cpu 23%` ｜ `memory 1.2G` ｜ `net 4.2 MB/s` ｜ `UTF-8`。
- CPU/内存/网络为当前会话的实时轻量指标（与 §11 资源面板同源，此处为常驻精简版）。

## 13. 新建连接 & 密码验证流程

### 13.1 新建连接弹窗（`oAHna`，500px）

- **头部**：标题「新建连接」+ `x`。
- **标签页（connTabs）**：`SSH` / `SFTP` / `Telnet` / `串口` 等协议切换。
- **表单主体（connBody）**：名称、Host、Port、用户名、认证方式（密码 / 密钥 / 跳板机）、所属分组、颜色/图标标注等。
- **页脚**：左=「测试连接」；右=`取消` / `连接`（accent）。

### 13.2 密码验证弹窗（两步式）

- **第 1 步（`oNZIM`，420px）**：头部标题 + 会话信息条（`bg-input`：主机图标 + `user@host`）+ 密码输入（可见切换）+ 「记住密码」勾选 + 页脚 `取消`/`连接`。
- **第 2 步（`twD13`）**：二次验证（2FA / OTP / 密钥口令 / 指纹主机校验）。信息条 + 输入 + `上一步`/`验证`。
- **主机指纹首次确认**：首次连接未知主机 → 弹「主机信任」确认（指纹 + 接受并保存/拒绝），关联 §15 主机信任中心。

**连接状态机**：`空闲 → 连接中(黄) → 认证 → 已连接(绿)` / 任一步失败 `→ 断开(红)` 并给出原因与重试。

## 17. 浮层管理与边界规则（交给 Agent 的通用约束）

1. **浮层层级**：设置/新建连接/密码=**模态弹窗**（居中 + 遮罩）；命令面板=**准模态**（遮罩，Esc 关）；文件传输/隧道/资源监控=**非模态悬浮**（无遮罩，可与主界面并存）。
2. **单例性**：命令面板、隧道面板、资源监控面板、文件传输组件各自全局单例；重复触发=聚焦已存在实例而非新建。
3. **屏幕/窗口边界避让**：所有跟随鼠标或锚点的浮层出界时自动翻转方向，始终完整可见。
4. **位置记忆**：文件传输组件、隧道面板的拖拽位置持久化。
5. **主题联动**：所有浮层跟随全局 dark/light 主题实时切换。
6. **资源面板 hover 计时**：进入标签名开始计时 400ms，中途移出即取消；移出面板与标签的合并区域后再关闭（防抖 ~150ms），避免"移向面板途中闪灭"。
7. **传输/隧道与会话生命周期**：会话断开时，其隧道置暂停、其传输任务暂停并提示；会话关闭前若有进行中传输，二次确认。
8. **空态**：无会话时终端区显示引导（新建连接/最近连接）；无传输时不渲染传输组件；无隧道时隧道面板显示"暂无隧道 + 新建"。
