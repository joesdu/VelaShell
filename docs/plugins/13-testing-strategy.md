# 13 · 测试策略

插件系统横跨进程边界,测试分四层;遵守既有测试纪律:UI 测试类严禁自
建 App/自行 StartNew/Dispose 会话(velashell-tests 既有教训),插件系
统新增测试全部接入现有测试基建。

## 1. 分层

### L1 纯逻辑单测(最大占比,无进程/无 IO)

- 生命周期状态机全迁移路径(M-3)
- manifest 校验器 20+ 坏例(M-1)
- `when` 表达式求值器(M-4)
- Broker 权限矩阵:每危险权限 × 七种授权状态(B-2)
- VelaUI diff 器:树变换 → 最小 patch(U-7)
- scope 匹配(路径规范化边界:`..`、尾斜杠、大小写)

### L2 协议契约测试(双端进程内回环,不起真进程)

- Pipe/UDS 换成内存 duplex 流,宿主端点 + SDK 代理直连
- 每个能力域:正常/权限拒绝/取消/中途断连四象限(C-8)
- DTO 二进制兼容:MessagePack 整数键快照 + PublicApiAnalyzer,apiLevel
  内破坏性变更 CI 红(P-3)
- 协议模糊:乱序/超长/非法载荷/握手前调用(P-7)

### L3 真进程集成测试

- 固定一组**测试插件**(仓库内,随 CI 构建):
  `wellbehaved`(全功能正常)/ `crasher`(激活即崩)/ `hanger`(堵死
  线程不回 ping)/ `hog`(吃内存)/ `chatty`(RPC 风暴)/
  `malformed`(协议乱字节)
- 场景:拉起/握手/激活耗时基线;杀进程后宿主状态机流转;心跳失联
  处置;退避重启到 Faulted;停机限时强杀;孤儿自杀(杀宿主,断言
  插件进程 2s 内消失)
- 混沌验收(H-8):故障插件运行期间,宿主 UI 线程无阻塞(帧探针)、
  同机第二个插件 RPC 延迟无劣化
- 三平台矩阵跑(Win/macOS/Linux——管道/UDS、Job Object、进程语义
  差异都在这层暴露)

### L4 端到端(官方示例即测试)

- 三个官方示例插件的冒烟脚本:安装 → 授权(测试模式下预授权注入,
  不弹 UI)→ 核心场景操作 → 卸载清理断言
- 远程能力对接现有 docker-compose.test.yml 的 SSH 测试容器
- 升级/回退/签名篡改/吊销四个分发场景(D-2/D-3)

## 2. 专项

- **性能基线**(接 05/08 验收):空调用延迟、流吞吐、VelaUI 1000 行
  列表滚动 IPC 流量、激活 P95;数值入 CI 趋势告警(参考 perf-pass 的
  基线纪律:多次采样、剔除启动漂移)
- **泄漏**:激活/停用循环 ×100 后宿主内存回到基线 ±10%;ALC 热重载
  ×50(dev 模式)PluginHost 内存有界
- **五语言**:NLS 缺漏 CI 检查(validate 已含);授权对话框/管理页
  文案入现有五语平价流程
- **安全**:红队脚本集(X-1)按威胁清单编号归档,回归可重放

## 3. CI 编排

| 触发 | 范围 |
| --- | --- |
| 每 PR | L1 + L2(分钟级) |
| 每 PR(标签 plugins) / 每夜 | L3 三平台 + L4 |
| 每夜 | 性能基线 + 泄漏 |
| 发布前 | 全量 + 红队回归 + 手动验收清单 |

## 4. 开发计划(本分项)

| 任务 | 说明 | 依赖 | 估算 |

| Q-1 | 测试插件六件套 + 构建接线 | H-1 | 2d |

| Q-3 | L3 进程测试基建(进程收割保障、三平台 CI 矩阵) | Q-1 | 3d |

带一条落在 L1–L4 某层的回归测试。
