# 10 · 打包、签名与分发

## 1. 签名与来源信任

在 v1 无 OS 沙箱的前提下(见 06/12),"用户装了什么、来自谁"是安全
模型的支柱,因此签名从第一版就是**强制**的。

方案:**开发者密钥签名 + 首次信任(TOFU)+ 官方源背书**,分三档:

| 信任档 | 含义 | UI 表现 |
| --- | --- | --- |
| 官方插件 | VelaShell 团队密钥签名 | 蓝标"官方" |
| 已验证发布者 | 密钥在官方源注册且通过发布者验证(GitHub 账号绑定等轻量验证) | 绿标"已验证:acme" |
| 未验证 | 有效自签名但发布者未注册 | 黄标警示;安装需二次确认("仅在信任来源时安装") |

机制:

- 签名格式:对包内全部文件的 SHA-256 清单(`SIGNATURE` 内含 manifest
  hash 树 + Ed25519 签名 + 公钥 + 可选官方源反签)。
- 无签名/签名损坏的包**拒绝安装**(开发模式 dev 装载除外,UI 明示 DEV)。
- **同 id 升级必须同公钥**(密钥连续性,防更新劫持);换钥需官方源
  背书流转。
- 宿主内置官方源根公钥;吊销列表随源索引下发(被吊销发布者的已装插件
  标红并默认禁用)。

## 2. 安装与更新管线

```text
来源(文件/URL/源页面)→ 下载到临时目录 → zip 安全解包(防 zip-slip)
 → 签名校验 → manifest 校验(schema/engines/platform/权限清单)
 → 权限预览页(逐条展示,危险权限红标;升级时高亮"新增权限")
 → 用户确认 → 落盘 installed/<id>/<ver>/ → 注册贡献点 → 完成(免重启)
```

- 更新检查:后台按源索引对比版本(默认每日;可关);更新沿用 03 §7
  的原子版本指针切换,失败回滚。
- 自动更新默认**关**(运维工具的保守取向:目标机器上意外变更是事故
  源);提供"仅安全更新自动装"(源索引标记 security 位)选项。
- 降级:管理页允许回退到本机缓存的上一版本(排障出口)。

## 3. 插件源(Registry)

分阶段,避免过早自建服务:

**阶段 A(v1)——静态索引源**:一个托管在 GitHub(Pages/Release)的
签名 JSON 索引,宿主可配置多个源(企业内网可自建同格式源,满足内网
分发场景):

```jsonc
// index.json(整体带官方签名)
{ "version": 1, "generatedAt": "...",
  "plugins": [ { "id": "acme.image-viewer", "latest": "1.2.0",
      "versions": [ { "version": "1.2.0", "engines": {"velaShell": ">=0.2.0", "apiLevel": 1},
          "url": "https://.../image-viewer-1.2.0.vpx", "sha256": "...",
          "publisherKey": "ed25519:...", "permissions": ["remote.files.read"],
          "security": false } ],
      "displayName": {"en": "...", "zh-Hans": "..."}, "description": {...},
      "icon": "https://...", "verifiedPublisher": true } ],
  "revokedPublishers": ["ed25519:..."] }
```

**阶段 B——官方市场**:动态服务(搜索、下载计数、评分、发布者门户、
自动扫描),仅当生态起量再投入;索引格式向前兼容阶段 A。

## 4. 应用内插件管理页

设置区新增"插件"主页面:

- **已安装**:卡片列表(图标/名称/发布者信任标/版本/状态色点);行内
  操作:启用/禁用/卸载/重启/查看日志;展开:权限、最近活动(审计)、
  资源占用(内存曲线)、不体面退出计数。
- **浏览**:聚合各配置源的索引,搜索/分类;详情页 = README 渲染 +
  权限清单 + 版本历史。
- **状态可视**:Faulted 红、无响应黄、运行绿、未激活灰——与 04 状态机
  一一对应。
- 五语言:页面文案入现有 resx 五语平价流程;插件自述文案用源索引的
  多语言字段,缺失回退英文。
