# 隧道功能规划（Plan）

> 更新于 2026-07-12。原"待实现"1~4 项均已落地；剩余迭代项见文末。

## 已实现

- 本地转发（-L）、远程转发（-R）、**动态转发（-D / SOCKS5）**三种类型，创建时下拉选择（动态转发自动隐藏目标主机/端口输入）
- 远程主机从资源管理器已保存的 SSH 会话中选择（服务端口手动填写）
- 隧道行内操作：停止 / 启动 / 删除（删除活动隧道时先自动停止）
- 表单"取消"清空重置；创建失败错误内联显示
- **隧道配置持久化**：按服务器分文档存入 SonnetDB `tunnels` 集合（docId = profileId，经 `IAppDataStore`）；应用重启后恢复为"已停止"状态，可一键启动（每次运行每台服务器只恢复一次，恢复失败不影响面板）
- **独立承载会话**：隧道生命周期独立于终端标签——未连接时创建/启动隧道会自动建立仅隧道用途的后台连接；该服务器最后一条隧道删除后后台连接自动断开
- **会话断开联动（基础）**：状态时钟检测后台会话掉线，把其隧道条目标记为"已停止"
- **多端同步**：隧道配置随「连接配置」范围纳入 Gist 云同步（plan.md §13.C）

## 待实现（后续迭代）

1. **断线自动恢复**：承载会话重连后可选自动重建其隧道（当前掉线只标记为已停止，需手动启动）。
2. **流量统计**：每条隧道的连接数 / 累计字节数，显示在行内（设计稿 fuXS7 预留了说明行）。
3. **端口冲突预检**：创建本地转发前检测本地端口占用，给出明确错误而非底层异常。
