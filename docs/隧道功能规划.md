# 隧道功能规划（Plan）

> 更新于 2026-07-07。已完成部分见下"已实现"；其余为后续迭代项，按优先级排序。

## 已实现

- 本地转发（-L）与远程转发（-R）两种类型，创建时下拉选择
- 远程主机从资源管理器已保存的 SSH 会话中选择（服务端口手动填写）
- 隧道行内操作：停止 / 启动 / 删除（删除活动隧道时先自动停止）
- 表单"取消"清空重置；创建失败错误内联显示
- 面板随活动标签的会话绑定（隧道建立在当前会话的 SSH 连接上）

## 待实现（后续迭代）

1. **隧道配置持久化**：把已创建的隧道配置存入 SonnetDB 文档集合（嵌入式，`%LocalAppData%/VelaShell/sonnetdb`，经 `IAppDataStore`），应用重启后在面板中恢复为"已停止"状态，可一键启动。
2. **独立承载会话选择**：目前隧道固定建立在当前活动标签的 SSH 会话上；后续允许在创建时选择任一已保存会话作为跳板（未连接时自动建立仅隧道用途的后台连接）。
3. **会话断开联动**：承载会话断开时把其隧道标记为失效并提示；会话重连后可选自动恢复隧道。
4. **动态转发（-D / SOCKS5）**：第三种类型，本地 SOCKS 代理。
5. **流量统计**：每条隧道的连接数 / 累计字节数，显示在行内（设计稿 fuXS7 预留了说明行）。
6. **端口冲突预检**：创建本地转发前检测本地端口占用，给出明确错误而非底层异常。
