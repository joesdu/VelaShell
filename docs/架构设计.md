# VelaShell 架构设计（工程化重构蓝图 v2）

> 本文基于三份输入交叉分析后编写：
> 1. 设计稿 `VelaShell-zh.pen`（通过 Pencil MCP 读取的节点树、设计 token、截图）；
> 2. 交互规格 `docs/交互与界面规格.md`（视觉与交互唯一基准）；
> 3. **现有代码库的逐文件走查**（6 个 src 项目 + 6 个测试项目，~129 个 .cs / ~22 个 .axaml）。
>
> 目标：在**不推倒现有可用能力**（自研 VT 引擎、SSH、Dock 分屏、命令面板、主题/i18n 骨架、359 个通过的测试）的前提下，给出一套能支撑你列出的 26 项目标的**工程化架构**，并把你反馈的每个卡顿/缺陷都对应到**根因**与**架构级修复方案**。
>
> 更新时间：2026-07-07。

## 0. 结论摘要（TL;DR）

- **分层不用重做**：现有 6 项目切分（App / Presentation / Controls / Terminal / Core / Infrastructure）与依赖方向基本正确，本文只做**职责收敛**与**新增 4 个贯穿性子系统**，不新增项目。
- **性能是头号工程问题**（你的 #5/#16）。卡顿的根因**不在 SSH，而在渲染管线**：`SshTerminalBridge` 对每个 4KB 读块都 `Dispatcher.UIThread.InvokeAsync` 并触发一次全屏 `InvalidateVisual`，而 `PulseTerminalControl.Render` **每帧、每个单元格都 new 一个 `FormattedText`**（120×32 ≈ 3840 次/帧的文本排版分配）。解决它需要一条**「读→合批→限帧→缓存字形」的输出管线**（见 §4.1），这是本次架构的核心。
- **卡死类缺陷**（#17 连接卡、#18 关闭卡）的根因是**同步阻塞 UI 线程**：`SshTerminalBridge.Dispose()` 在 UI 线程 `_readTask.Wait(2s)` + 同步 `ShellStream.Dispose()`。方案是**会话生命周期与传输层解耦 + 后台拆卸队列**（§4.3）。
- **中文输入炸 htop（#14）** 的根因是**未实现 IME 合成态处理**：合成候选期间 `OnKeyDown` 仍把方向键/回车/ESC 编码成字节发给 PTY。方案是接入 Avalonia `TextInputMethodClient`，合成期间**屏蔽按键编码**并把候选窗定位到光标（§4.2）。
- **滚动条无效（#15）** 的根因是：XAML 里的 `ScrollBar` 只做**单向只读展示**，真正的滚动在控件内部靠滚轮 `_scrollOffset`；且 `Updated` 事件**无条件把 `_scrollOffset` 归零**，任何后台输出都会把你弹回底部。方案是引入**视口/滚动模型 + `IScrollable`**（§4.1.4）。
- **SFTP（#22）** 其实**核心层已实现**（`SftpService`/`TransferManager` 完整），但**未注册进 DI、VM 被塞了 null**。属于「接线 + 补 UI」，不是从零（§4.8）。
- **UI 需按设计稿重建（#26）**：现有 `VelaShell.Controls` 只有 1 个控件 + token 字典；需扩成真正的**自定义控件库**，逐帧对齐设计稿（§4.9 / §6）。

优先级：**P0 = 渲染管线 + 生命周期解耦 + IME + 滚动**（直接消除你反馈的卡顿/崩溃）；**P1 = SFTP 接线 + UI 控件库重建 + 命令注册表/浮层管理器**；**P2 = 高级面板与设置全页**。

## 3. 分层与依赖方向（收敛后）

```
                         ┌───────────────────────────────┐
                         │  VelaShell.App (WinExe)        │  组合根/DI、窗口壳、Dock、
                         │  Views(axaml) + App 级 VM      │  浮层宿主、命令绑定
                         └───────────────────────────────┘
             ┌───────────────┬───────────────┬───────────────┐
             ▼               ▼               ▼               ▼
   ┌──────────────┐ ┌──────────────┐ ┌──────────────┐ ┌──────────────────┐
   │ Presentation │ │  Controls    │ │  Terminal    │ │ Infrastructure   │
   │ 跨层 VM/工作流│ │ 自定义控件+  │ │ VT 引擎+渲染 │ │ SSH/SFTP/隧道/   │
   │              │ │ token/图标   │ │ +输入+管线   │ │ 存储 实现 + DI   │
   └──────────────┘ └──────────────┘ └──────────────┘ └──────────────────┘
             └───────────────┴───────┬───────┴───────────────┘
                                     ▼
                         ┌───────────────────────────────┐
                         │  VelaShell.Core（叶子）        │  领域模型、契约接口、
                         │  模型/契约/服务/i18n/持久化    │  纯逻辑服务（无 Avalonia 依赖）
                         └───────────────────────────────┘
```

**硬性规则**
- `Core` 不引用 Avalonia、不引用任何上层（保持可测、可复用）。
- `Terminal` 只依赖 `Core`（不依赖 Presentation/App）。
- `Infrastructure` 只依赖 `Core`（SSH.NET 只在这里出现；App 通过**按类型名匹配**处理 SSH.NET 异常，避免直接引用——这条现有做法很好，保留）。
- `Controls` 只依赖 Avalonia + `Core`（共享 UI 契约），**不依赖 App**：这样运行时换肤/换色只动 token 字典即可。
- **所有跨层交互走 `Core` 中定义的接口**，实现放 `Infrastructure`/`App`，用 DI 组装。

## 4. 贯穿性核心子系统（工程化重点）

以下 10 个子系统是本次架构的「肉」。每个都标注：**归属项目 / 解决的需求 / 关键接口 / 落地要点**。

### 4.1 终端渲染与输出管线 ★核心（需求 #5、#16、#10、#15）

归属：`VelaShell.Terminal`。这是消除卡顿的关键，拆成四段流水线：

```
ShellStream(后台读)  →  [1 输出通道/合批泵]  →  TerminalEmulator.Feed(合批)
                                                        │  Updated
                                                        ▼
                                   [2 帧调度器 限 60fps]  →  RequestFrame
                                                        │
                                                        ▼
                        [3 字形/行缓存 + 脏行渲染]  →  DrawingContext
                                                        ▲
                                   [4 视口/滚动模型]  ── 提供可视区与偏移
```

**[1] 输出合批泵（解决「读一次刷一次」）**
- `SshTerminalBridge` 的读循环**不再**对每个 4KB 块 `InvokeAsync(Feed)`。改为把字节写入一个 `System.Threading.Channels.Channel<byte[]>`（或环形缓冲）。
- UI 线程上一个**合批泵**（`DispatcherTimer` 或 `Dispatcher.UIThread.Post` 自泵）在每帧边界把通道里**攒下的所有块拼成一次 `Feed`**。高吞吐场景（`apt upgrade`、`cat 大文件`、进度条刷屏）从「每 4KB 一次跨线程 + 一次全屏重绘」降为「每帧一次」。
- 好处：`apt`/`yum` 的 `\r` 原地重绘进度条（你的 #10）在合批后天然平滑；不会因为刷屏把 UI 线程打满。

**[2] 帧调度器（解决「无限帧 / 抖动」）**
- 把 `Updated → InvalidateVisual` 改成 `Updated → 置脏 + 请求下一帧`。用固定节流（默认 60fps，可在设置里调）合并多次脏。空闲时 0 重绘。
- 光标闪烁用**独立低频计时器**（不触发全屏重绘，只重绘光标行）。

**[3] 字形/行缓存 + 脏行渲染（解决「每格 new FormattedText」——最大性能杀手）**
- 现状 `Render` 里每格 `new FormattedText(...)`；**替换为**按「(字形串, 前景, 样式) 段」构建 `GlyphRun`/缓存的 `TextLayout`，并**按行缓存**排版结果，行内容未变则直接复用。
- 结合 `TerminalRow` 增加**脏标记**（emulator 写入时置脏），`Render` 只重排/重画脏行，未变行走缓存位图/几何。
- 等宽假设下可进一步做「按样式分段的 run 合并」：一行里同前景/同 flag 的连续格合并成一个 run，单行 draw 调用数从 O(列数) 降到 O(样式段数)。
- CJK 双宽已由 `CharWidth`/`IsWideTrailing` 处理，保留；缓存 key 需含 flags（粗体/斜体/下划线）。

**[4] 视口/滚动模型（解决 #15 滚动条无效）**
- 新增 `TerminalViewport`：暴露 `TotalRows` / `ViewportRows` / `ScrollOffset`（可读写、可绑定）。
- `PulseTerminalControl` 实现 Avalonia `IScrollable`（或在 `TerminalTabView` 里放一个真正双向绑定的 `ScrollBar`），滚动条**拖动能驱动 `ScrollOffset`**，而不是现在的只读展示。
- **关键修复**：`OnEmulatorUpdated` 不再无条件 `_scrollOffset = 0`。改为「**仅当当前已在底部**才吸底」；用户上滚查看历史时，后台输出**不打断**。
- 滚动历史容量：`MaxScrollback` 默认从 10000 提到**可配置（建议默认 50000，设置里可调）**；alt-screen 不计历史（现状正确）。

**验收信号**：`cat` 一个上万行文件不掉帧；`htop`/`top` 常驻刷新 CPU 占用低；上滚历史时输出不把你弹回底部；拖动滚动条可浏览全部历史。

### 4.2 输入管线与 IME（需求 #14 中文、#20 复制粘贴键、#8 输入体验）

归属：`VelaShell.Terminal`（控件层）。

- **IME 合成态（#14 htop F3 崩溃根因）**：`PulseTerminalControl` 实现 `Avalonia.Input.TextInput.TextInputMethodClient`，向框架暴露光标矩形（候选窗定位到终端光标处）。**核心修复**：维护 `IsComposing` 状态，**合成进行中时 `OnKeyDown` 不调用 `InputEncoder.Encode`**（方向键/回车/ESC 用于选候选，不能漏给 PTY）；合成提交后由 `OnTextInput` 一次性发 UTF-8。这样在 htop 搜索框输中文不会再把 ESC 发进去导致退出。
- **复制粘贴（#20）**：现状已有 `Ctrl+Shift+C/V` + 右键粘贴 + 括号粘贴。**新增 `Shift+Insert` 粘贴**；**新增「选中即复制」**（选区释放 `OnPointerReleased` 时按设置自动写剪贴板，满足你 #8「选中复制而非 Ctrl+C」）。`Ctrl+C` 保持发送中断信号（SIGINT）。
- **多行粘贴（#8）**：括号粘贴模式已支持；对非括号模式做 `\r\n→\r` 归一（现状已做），并对超大粘贴给一次确认提示（防误贴）。
- **收敛双路径**：删除 `TerminalTabView.axaml.cs` 里重复的输入/选区逻辑，全部走控件；code-behind 只把**未被终端消费的全局快捷键**上抛给命令系统（§4.6）。

### 4.3 会话生命周期与连接编排（需求 #17 连接不卡、#18 关闭秒关、#19 原标签重连）

归属：契约在 `Core`，编排在 `Presentation`/`App`。**把「标签/VM/屏缓冲」与「传输层（SshClient/ShellStream/Bridge）」解耦**。

引入 `TerminalSession` 状态机（`Core.Models`）：
```
Idle → Connecting → Connected → Disconnected → (Reconnecting → Connected)
                 ↘ Failed ↗
```

- **#17 连接不卡**：`ConnectProfileAsync` 双击后**立即创建标签页并进入 `Connecting` 态**（终端区显示「连接中」覆盖层），SSH 连接与 `CreateShellStream` 全程在**后台线程**；成功后把 `ShellStream` 挂到已存在的 emulator 上并转 `Connected`。UI 线程一刻不阻塞，10s 超时也只影响那个标签的覆盖层。
- **#18 关闭秒关**：关闭标签时**同步只做**「从 Dock/TabBar 移除 + 视觉消失」；把 `Bridge.Dispose()`/`ShellStream.Dispose()` 丢进一个 **`ISessionTeardownQueue`（后台单线程队列）**。彻底移除现在 `Dispose()` 里的 `_readTask.Wait(2s)`（改为带超时的后台等待）。用户瞬间看到标签关闭，断连在后台完成。
- **#19 原标签重连**：`Disconnected` 态时保留 emulator 的屏缓冲与标签，显示断开覆盖层（§7 `ZufZw`）。绑定 **回车 / `Ctrl+R`** → `ReconnectCommand`：复用同一 `TerminalSession`/emulator/标签，仅**重建 `ShellStream + SshTerminalBridge`**。`exit`/`reboot` 触发的 EOF（读循环 `bytesRead==0`）应转成 `Disconnected` 事件而非直接销毁标签。支持指数退避自动重连开关（`ReconnectAttempts` 字段已在，接上驱动即可）。

### 4.4 主题与强调色（需求 #2 深浅色、#3 跟随系统+强调色实时）

归属：`Core`（服务）+ `App`（Avalonia 绑定）+ `Controls`（token）。

- token 字典已存在（`Controls/Themes/{PulseTokens,PulseShellTokens}.axaml` + `App/Themes/{Dark,Light}Theme.axaml`），全部走 `DynamicResource`，运行时换肤基础已具备。
- **跟随系统实时（#3 缺口）**：`ThemeService` 增加 `System` 模式下订阅 `TopLevel.PlatformSettings.ColorValuesChanged`（Avalonia 提供 OS 明暗变化通知），OS 切换时把 `Application.RequestedThemeVariant` 置为 `Default` 并让 Fluent 跟随——**无需重启**。
- **强调色运行时覆盖（#3）**：新增独立「强调色层」——把 `accent`/`accent-dim`/`accent-text` 从主题字典中抽出为**可被用户覆盖的资源**，设置里改色 → `Application.Resources` 里替换这几个 key → 所有 `DynamicResource` 引用即时刷新。终端调色板（`TerminalPalette`）同步更新 accent 对应色。
- 设计 token 权威值以 `VelaShell-zh.pen`（本文 §附录 token 表）为准，**禁止硬编码颜色**（你的规格 §1）。

### 4.5 国际化（需求 #4 多语言、实时）

归属：`Core`（服务/资源）+ `App`（markup 扩展）。

- 保留 resx 卫星程序集（EN + zh-CN），`LocalizationService` 已实现 `SetLanguage`（设 `CurrentUICulture`）——**先把它注册进 DI**（现在漏了）。
- **实时切换（缺口）**：`{x:Static Strings.X}` 在加载时固化、不随语言变。改用一个 **`LocalizeExtension`（markup extension）** 绑定到 `ILocalizationService` 的 `CurrentLanguage`（`INotifyPropertyChanged`/Observable），切语言时所有绑定文本即时刷新，无需重启。
- **清理硬编码**：命令面板分类「会话/命令」、窗口按钮 tooltip「最小化」等散落的中文，迁进 resx（键名规范 `Menu.Session`、`Tab.Close` 等），与命令注册表（§4.6）共用键。

### 4.6 命令注册表（统一 菜单 §4A ‖ 命令面板 §8 ‖ 快捷键 §16）

归属：`Presentation`（注册表）+ `App`（绑定）。规格 §4A.1 明确要求「菜单项 / 命令面板 / 快捷键三者共享同一套命令注册表」。

- 新增 `ICommandRegistry`：每条命令 = `{ Id, 本地化标题键, 图标(PathIcon key), 默认快捷键, CanExecute, Execute, 分组, 上下文(Global/Terminal/File) }`。
- **单一数据源**驱动三个消费者：
  - **菜单栏**（§4A.1 会话/编辑/操作/搜索/工具/帮助）由注册表按分组渲染；
  - **命令面板**（§8）对注册表做模糊子序列搜索（现有 `CommandPaletteViewModel` 改为从注册表取数）；
  - **快捷键**（§16）由 `KeyboardShortcutService`（已存在，需进 DI）把键位映射到命令 Id。
- 终端工具栏那些 no-op 命令（`Search/Copy/Split/Broadcast/Tunnel/QuickCommands`）改为**指向注册表命令**，一处实现处处生效。

### 4.7 浮层 / 悬浮面板管理器（规格 §17 通用约束）

归属：`App`（宿主）+ `Presentation`（VM）。规格 §17 规定了模态性/单例/边界避让/位置记忆/主题联动等**统一规则**，必须有一个中枢来落实，而不是每个面板各写一套。

新增 `IOverlayService`：
- **三类层级**：模态弹窗（设置/新建连接/密码，居中+遮罩）、准模态（命令面板，遮罩+Esc）、非模态悬浮（文件传输/隧道/资源监控，无遮罩、可与主界面并存）。
- **单例聚焦**：命令面板、隧道、资源监控、文件传输组件全局单例，重复触发聚焦已存在实例。
- **边界避让**：跟随鼠标/锚点的浮层（资源监控 §11、隧道 §10）出界自动翻转方向。
- **位置记忆**：文件传输组件、隧道面板拖拽位置持久化（存 `AppState`）。
- **主题联动**：浮层跟随全局明暗（因为都走 `DynamicResource`，天然满足）。
- **hover 计时器**（§4B.4/§11）：标签名 hover>400ms 唤出资源监控、150ms 防抖淡出，统一在此实现避免各处重复。

### 4.8 SFTP 与文件传输（需求 #22，规格 §6 文件浏览器 / §9 传输组件）

归属：服务在 `Core`（**已实现**）+ 传输编排在 `Presentation` + UI 在 `App`/`Controls`。

- **接线优先**：`SftpService`/`ITransferManager` 注册进 DI（现在 `MainWindowViewModel` 里给 `FileBrowserViewModel`/`FileTransferViewModel` 传的是 `null`）。`ISftpClientWrapper` 从已连接的 `SshConnectionService` 复用同一会话凭据。
- **文件浏览器（§6）**：右区下部可折叠面板，面包屑路径 + 列（文件名/大小/权限/时间）+ 排序 + 右键菜单（下载/上传/重命名/删除/chmod/复制路径/新建）+ 双击进目录/下载。首行 `..` 返回上级。
- **传输队列（§9）**：`TransferManager` 已支持并发上限 + 进度/速度/ETA。UI 侧做**右上角悬浮传输组件**：仅在有进行中任务时出现，头部为拖拽把手（走 §4.7 位置记忆），多任务纵向堆叠 + 徽章计数，全部完成停留 ~3s 后**自动淡出**，失败标红 + 重试。
- **线程**：所有传输在后台线程（`TransferManager` 已如此），进度回调 `IProgress<TransferProgress>` 编组回 UI，满足你 #5「后台传输时界面依旧流畅」。会话断开 → 传输暂停（§17.7）。
- 拖拽上传/下载（§6）：本地文件拖入 = 上传；列表项拖到本地 = 下载。

### 4.9 自定义控件体系与图标（需求 #1 大量自定义控件、#24 图标、#25/#26 严格按设计稿）

归属：`VelaShell.Controls`（把它做成真正的设计系统库）。规格 §26 明确：现有 UI 是初版、**需按设计稿用自定义控件重构**，不用 Avalonia 默认控件。

- 现状 `Controls` 只有 `StatusMetricChip` + token 字典。**扩为控件目录**（templated `TemplatedControl`，`Themes/*.axaml` 放 `ControlTemplate`），每个控件严格对齐设计稿的 token/尺寸/圆角/间距（规格 §1.3）。控件清单见 §6 映射表。
- **图标（#24）**：统一用 `PathIcon` + 一个 `Icons.axaml` 资源字典（`StreamGeometry`）。设计稿用 **lucide**，把用到的 lucide 图标路径收进字典（key 如 `Icon.Plus`、`Icon.Route`、`Icon.Search`），控件用 `<PathIcon Data="{StaticResource Icon.Route}"/>`。尺寸 11–16（规格 §1.3）。
- **字体**：`font-mono = JetBrains Mono`（终端/主机名/端口/路径/快捷键/标签名），`font-ui = Inter`（菜单/按钮/说明）。JetBrains Mono 建议**内嵌**并配置 CJK 回退（`Microsoft YaHei`/`Noto Sans CJK`），修掉「未指定中文回退字体」的坑（plan.md §10）。
- **dark 优先**：规格 §25 说以深色为准、浅色仅大致参考——控件模板先把深色打磨到位，浅色用 token 覆盖跟随。

### 4.10 语义高亮 / 链接·错误识别（需求 #9）

归属：`VelaShell.Terminal`（渲染附加层）。

- 新增 `ISemanticMatcher`：对**已提交的行文本**（非逐字节）跑一组正则/规则，产出「span + 类型（URL/错误/警告/路径/IP）」。默认规则：URL（`https?://…`）、错误关键字（`error/failed/fatal/panic`）、警告（`warn`）。
- 渲染层在 §4.1[3] 的行缓存之上叠加**语义样式**（URL 加下划线 + `info` 蓝、错误 `error` 红等，颜色走 token）。URL 可点击（`OSC 8` 超链接现状已解析，规则匹配作为兜底）。
- 规则**可配置**（设置里开关/自定义正则），与终端 buffer 解耦，不污染 VT 状态。

### 4.11 持久化与安全

- **全面采用 SonnetDB(嵌入式)**（https://github.com/IoTSharp/SonnetDB ,`SonnetDB.Core`,数据目录 `%LocalAppData%/VelaShell/sonnetdb`),替代原 JSON/LiteDB 方案;旧 JSON 文件首次运行一次性导入。接口在 `Core`,实现在 `Infrastructure/Persistence/SonnetDb*`,共享单例 `SonnetDbEngine`(退出时 Dispose 刷 WAL)。
  - **文档集合**(JSON 文档,业务/配置数据):`session_groups`、`session_profiles`($.groupId 索引)、`app_config`(settings/state 单文档)、`known_hosts`、`ui_config`(界面配置)、`quick_commands`。
  - **时序 measurement**(时间相关数据):`conn_history`(连接历史,tags: profile_id/host/username,fields: name/group_name/port/success/duration_ms,支撑侧边栏“最近连接”)、`audit_log`(审计,tags: category/action/profile_id,fields: detail;会话录制/命令历史等后续同样走时序)。
- **Dock 布局持久化**：Dock 的 `Layout` 序列化后存入 `app_config`/`ui_config` 文档集合,重启恢复分屏/标签（plan.md 待办）。
- **密码加密（已落地）**：`ISecretProtector`（`Core` 接口）+ `AesSecretProtector`（AES-256-GCM,密钥文件 `%LocalAppData%/VelaShell/secret.key`）,会话密码/私钥口令加密后写入 SonnetDB;历史明文在读取时兼容、下次保存时自动加密。未勾选“记住密码”时凭据不落盘。

## 7. 线程模型总览（落实需求 #5）

| 线程/上下文 | 职责 | 禁止 |
|---|---|---|
| **UI 线程（Dispatcher）** | 渲染、输入、布局、VM 绑定、emulator.Feed（合批后） | ❌ 任何网络/磁盘同步等待；❌ `Task.Wait`/`.Result`；❌ `ShellStream.Dispose` |
| **SSH 读线程**（每会话 1 个 `Task`） | `ShellStream.ReadAsync` → 写入 Channel | ❌ 直接碰 Avalonia 对象 |
| **SSH 写**（fire-and-forget async） | 用户输入 `WriteAsync` | — |
| **传输线程池**（`TransferManager`，并发上限） | SFTP 上传/下载，进度经 `IProgress` 编组回 UI | — |
| **拆卸队列**（后台单线程） | Bridge/ShellStream/Client 的 Dispose（§4.3 #18） | ❌ 阻塞 UI |
| **合批泵/帧调度**（UI 线程计时器） | 攒块合并 Feed + 限帧重绘 | — |

原则：**UI 线程只做「快而确定」的工作**；一切可能阻塞（连接、断开、传输、DNS、握手）都在后台，UI 用状态机/覆盖层反映进度。

## 10. 关键决策与风险

| 决策点 | 建议 | 风险/备注 |
|---|---|---|
| .NET 11 vs 10 | 按你要求默认 **net11**；生产可回退 net10（LTS，一行改动） | net11 preview 稳定性；需实机全量测试 |
| 渲染重写 vs 微调 | **重写渲染热路径**（字形缓存/脏行/合批）而非小修 | 引擎/屏缓冲不动，只改 `Rendering/`，风险可控、测试覆盖已在 |
| IME | 实现 `TextInputMethodClient` | 各平台 IME 行为差异，需 Win/macOS/Linux 各验一次 |
| 双输入路径 | 收敛到控件，删 code-behind 重复逻辑 | 小心别回归复制/选区行为，靠测试兜底 |
| UI 重建 | 按设计稿逐帧、逐控件替换，P0 不动 UI 只修内核 | 避免「大爆炸重写」，与内核修复并行推进 |
